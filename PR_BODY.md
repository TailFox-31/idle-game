## 요약

현재 자동 전투 프로토타입에 플레이어 스킬 시스템을 바로 구현하기보다, 보스 메커닉 층을 이후 플레이어도 재사용할 수 있도록 1차 설계 문서를 추가했습니다.

핵심 방향은 보스 전용 `BossMechanicRuntime` 계층을 공통 전투 메커닉 모델로 일반화하고, 플레이어 스킬을 그 위에 얹는 것입니다. 이를 기준으로 MVP 후보 스킬, 자원 모델, 저장/UI 영향, 구현 순서를 정리했습니다.

## 이번 PR에 포함된 내용

- `docs/player-skill-design.md` 추가
- 현재 전투 루프와 플레이어 스킬의 결합 지점 정리
- 보스 메커닉 재사용 관점의 MVP 스킬 후보 제안
- 저장 데이터/UI/단계별 구현 순서/리스크 정리

## MVP 제안

1차 구현 권장 스킬은 아래 4종입니다.

- Burst: `WindUpBurst` 재사용
- Frenzy: `FrenzyWindow` 재사용
- Guard: `GuardRecovery` 재사용
- Last Stand: `EnrageThreshold` 재사용

`ReflectWindow` 기반 Reflect는 설계 문서에는 포함했지만, 처리 순서와 피드백 난도를 고려해 MVP 다음 후보로 분리했습니다.

## 설계 결론

- 자원 모델은 우선 `개별 쿨다운 + 활성 시간` 기반이 가장 적절합니다.
- 전투 중 runtime 상태는 저장하지 않고, 해금/장착 같은 진행도 데이터만 저장하는 방향이 안전합니다.
- 구현은 공통 메커닉 추출 -> 플레이어 스킬 정의 추가 -> 전투 루프 연결 -> UI 노출 -> 저장/밸런싱 순서로 진행하는 것이 좋습니다.
